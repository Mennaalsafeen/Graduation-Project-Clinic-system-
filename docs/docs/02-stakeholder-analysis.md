# Stakeholder Analysis

## Stakeholders Overview
Stakeholders are individuals or groups who are involved in or affected by the Clinic Appointment Management System.

## Identified Stakeholders

### 1. Admin
- Responsible for managing the system.
- Can add, update, and delete users.
- Manages appointments and system settings.

### 2. Doctor
- Views assigned appointments.
- Manages daily schedule.
- Reviews patient appointment details.

### 3. Patient
- Registers into the system.
- Books, updates, and cancels appointments.
- Views appointment history.

### 4. System Developer
- Responsible for developing and maintaining the system.
- Implements features and fixes issues.

### Stakeholder Needs

| Stakeholder | Needs |
|------------|------|
| Admin | Full control over system management |
| Doctor | Easy access to schedule and appointments |
| Patient | Simple and fast booking system |
| Developer | Maintainable and scalable system |

## Conclusion
Understanding stakeholders helps ensure that the system meets the needs of all users and provides an efficient and user-friendly experience for managing clinic appointments.
