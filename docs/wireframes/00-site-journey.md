# Site Journey

## 1. Guest Path

These pages are for users who do not have an account or have not logged in yet.  
The path includes:

- Home: The main page of the website, showing an overview of services or general information.  
- Login: The login page for registered users. Includes fields for email/username and password, plus a login button.  
- Register: The page for creating a new account. Contains fields for full name, email, password, phone number, and password confirmation.  
- Browse basic info: Users can view basic information about services or content without logging in.  
- [?] Any unclear pages: If there is a page in the design that is unclear, mark it with [?] for later clarification.  

---

## 2. Logged-in User Path

These pages are for users who have logged in. The path includes:

- Home: Same as the main page, but may show user-specific information.  
- Dashboard: Displays a summary of the user’s status, such as appointments or activities.  
- View appointments: Allows the user to see all appointments or services registered under their name.  
- Book appointment: Page to book a new appointment, with options to choose service, date, and time.  
- Logout: A button to log out of the account.  
- [?] Any unclear pages: Mark with [?] if any part of the wireframe is unclear.  

---

## 3. General Notes About the Site

- All pages should include a Navigation Bar at the top to make it easy to move between pages.  
- Every page should have a Footer showing rights and important links.  
- Login and registration pages should have clear fields and easily understandable buttons.  
- Landing pages should display key sections and clear buttons for registration and login.  
- This wireframe shows only the basic structure of the site; highly detailed aesthetic design is not required, the main goal is that all elements are clear.
